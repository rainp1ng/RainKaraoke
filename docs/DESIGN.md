# RainKaraoke 详细设计方案

## 一、技术架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        Tauri Application                        │
├─────────────────────────────────────────────────────────────────┤
│  Frontend (React + TypeScript)                                  │
│  ├── UI Components (播放器、队列、媒体库、控制面板)               │
│  ├── State Management (Zustand)                                 │
│  └── Audio Visualization (Web Audio API)                        │
├─────────────────────────────────────────────────────────────────┤
│  Tauri Core (Rust)                                              │
│  ├── Media Engine (ffmpeg-next + rodio)                        │
│  ├── Audio Router (多声道输出管理)                              │
│  ├── MIDI Handler (midir)                                       │
│  ├── VAD Module (webrtc-vad 或 vosk)                           │
│  └── Database (rusqlite + SQLite)                              │
├─────────────────────────────────────────────────────────────────┤
│  Native Dependencies                                            │
│  ├── FFmpeg (音视频解码)                                        │
│  ├── SQLite (元数据存储)                                        │
│  └── PortAudio/CoreAudio (音频I/O)                             │
└─────────────────────────────────────────────────────────────────┘
```

## 二、核心模块设计

### 2.1 媒体库管理模块

```
┌────────────────────────────────────────────────────────┐
│                  Media Library Module                   │
├────────────────────────────────────────────────────────┤
│ 功能：                                                  │
│ • 文件导入：拖拽/文件夹扫描/单个文件添加                │
│ • 元数据提取：自动从文件名/ID3标签提取信息              │
│ • 歌词解析：支持 LRC(时间轴) / KSC(逐字) / TXT(纯文本) │
│ • 标签管理：自动标签(歌手/风格) + 自定义标签            │
│ • 搜索过滤：全文搜索 + 多条件筛选                       │
│ • 批量操作：批量修改标签、删除、移动                    │
└────────────────────────────────────────────────────────┘

文件导入流程：
┌──────────┐    ┌──────────────┐    ┌───────────────┐
│ 选择文件 │───>│ 分析文件类型 │───>│ 提取元数据    │
└──────────┘    └──────────────┘    └───────┬───────┘
                    │                      │
                    v                      v
              ┌──────────┐          ┌───────────────┐
              │ 检测关联 │          │ 匹配歌词文件  │
              │ 文件     │          │ (同名.lrc/.ksc)│
              └────┬─────┘          └───────────────┘
                   │
                   v
         ┌─────────────────┐
         │ 自动关联：       │
         │ • 同名视频/音频  │
         │ • 原唱/伴唱文件  │
         │ • 歌词文件       │
         └─────────────────┘
```

### 2.3 过场音乐模块

```
┌─────────────────────────────────────────────────────────────┐
│                   Interlude Music Module                     │
├─────────────────────────────────────────────────────────────┤
│ 触发条件：                                                  │
│ • 歌曲播放结束                                              │
│ • 用户点击暂停                                              │
│ • 队列为空时                                                │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ Ducking 逻辑：                                              │
│                                                             │
│          ┌──────────────┐                                  │
│          │ VAD 检测线程  │                                  │
│          └──────┬───────┘                                  │
│                 │                                           │
│                 v                                           │
│         ┌───────────────┐                                   │
│         │ 音量 > 阈值?  │                                   │
│         └───────┬───────┘                                   │
│           Yes   │   No                                      │
│          ┌──────┴──────┐                                    │
│          v             v                                    │
│   ┌────────────┐ ┌────────────┐                            │
│   │ Duck 音量  │ │ 恢复音量   │                            │
│   │ (平滑过渡) │ │ (平滑过渡) │                            │
│   └────────────┘ └────────────┘                            │
│                                                             │
│ Ducking 参数：                                              │
│ • 阈值: 0.05 - 0.2 (可调)                                  │
│ • 降低比例: 10% - 30%                                      │
│ • 过渡时间: 100-300ms (平滑淡入淡出)                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 气氛组模块

```
┌─────────────────────────────────────────────────────────────┐
│                   Atmosphere Sound Module                    │
├─────────────────────────────────────────────────────────────┤
│ MIDI 映射：                                                 │
│                                                             │
│  MIDI键盘按键  ──────>  音效触发                            │
│  (Note On)           (播放/停止)                            │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  MIDI Key  │  Action     │  Mode                     │  │
│  ├────────────┼─────────────┼───────────────────────────┤  │
│  │  C3 (60)   │  掌声       │  One-shot (播放完自动停)  │  │
│  │  C#3 (61)  │  欢呼       │  One-shot                 │  │
│  │  D3 (62)   │  倒计时     │  One-shot                 │  │
│  │  D#3 (63)  │  气氛音乐   │  Loop (循环直到再按)      │  │
│  │  E3 (64)   │  笑声       │  One-shot                 │  │
│  │  ...       │  自定义     │  可配置                   │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ 优先级规则：                                                │
│                                                             │
│ 气氛组播放 ─────> 过场音乐暂停                              │
│ 气氛组停止 ─────> 过场音乐恢复 (如果之前在播放)            │
│                                                             │
│ 状态转换：                                                  │
│ IDLE ──(Note On)──> PLAYING ──(Note Off/End)──> IDLE       │
└─────────────────────────────────────────────────────────────┘
```

### 2.6 人声效果器链模块

```
┌─────────────────────────────────────────────────────────────┐
│                  Voice Effect Chain Module                   │
├─────────────────────────────────────────────────────────────┤
│ 架构设计：                                                   │
│                                                             │
│   麦克风输入 ──> [效果器链] ──> 输出设备                    │
│                    │                                        │
│                    v                                        │
│              ┌──────────┐                                   │
│              │ 效果器槽  │                                   │
│              └────┬─────┘                                   │
│                   │                                         │
│    ┌──────────────┼──────────────┐                         │
│    │              │              │                          │
│    v              v              v                          │
│ ┌──────┐     ┌──────┐     ┌──────┐                         │
│ │ 槽位1│     │ 槽位2│     │ 槽位3│  ... (最多8个槽位)       │
│ └──┬───┘     └──┬───┘     └──┬───┘                         │
│    │            │            │                              │
│    v            v            v                              │
│ [混响]      [合唱]       [EQ]                               │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ 支持的效果器：                                               │
│                                                             │
│ 1. 混响 (Reverb)                                            │
│    • Room Size: 房间大小 (0-100)                           │
│    • Damping: 阻尼 (0-100)                                 │
│    • Wet Level: 湿声比例 (0-100%)                          │
│    • Dry Level: 干声比例 (0-100%)                          │
│    • Pre-delay: 预延迟 (0-100ms)                           │
│                                                             │
│ 2. 合唱 (Chorus)                                            │
│    • Rate: 调制速率 (0.1-10 Hz)                            │
│    • Depth: 调制深度 (0-100%)                              │
│    • Mix: 混合比例 (0-100%)                                │
│    • Voices: 声音数量 (1-8)                                │
│    • Spread: 声像展开 (0-100%)                             │
│                                                             │
│ 3. 均衡器 (EQ) - 参数均衡                                   │
│    • 低频 (Low): 增益 ±12dB, 频率 20-200Hz                │
│    • 低中频 (Low-Mid): 增益 ±12dB, 频率 200-2kHz          │
│    • 高中频 (High-Mid): 增益 ±12dB, 频率 2k-8kHz          │
│    • 高频 (High): 增益 ±12dB, 频率 8k-20kHz               │
│                                                             │
│ 4. 压缩器 (Compressor)                                      │
│    • Threshold: 阈值 (-60 to 0 dB)                         │
│    • Ratio: 压缩比 (1:1 to 20:1)                           │
│    • Attack: 启动时间 (0.1-100ms)                          │
│    • Release: 释放时间 (10-1000ms)                         │
│    • Makeup Gain: 补偿增益 (0-24dB)                        │
│                                                             │
│ 5. 延迟 (Delay)                                             │
│    • Time: 延迟时间 (1-1000ms)                             │
│    • Feedback: 反馈 (0-90%)                                │
│    • Mix: 混合比例 (0-100%)                                │
│    • Ping-Pong: 乒乓延迟开关                               │
│                                                             │
│ 6. 去齿音 (De-Esser)                                        │
│    • Frequency: 中心频率 (2k-12kHz)                        │
│    • Threshold: 阈值 (-40 to 0 dB)                         │
│    • Range: 衰减范围 (0-24 dB)                             │
│                                                             │
│ 7. 激励器 (Exciter)                                         │
│    • Frequency: 中心频率 (2k-16kHz)                        │
│    • Harmonics: 谐波量 (0-100%)                            │
│    • Mix: 混合比例 (0-100%)                                │
│                                                             │
│ 8. 噪声门 (Noise Gate)                                      │
│    • Threshold: 阈值 (-80 to -20 dB)                       │
│    • Attack: 启动时间 (0.1-50ms)                           │
│    • Release: 释放时间 (10-500ms)                          │
│    • Range: 衰减范围 (0-80 dB)                             │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ 音频流处理：                                                 │
│                                                             │
│  输入源选择                                                  │
│  ┌────────────┐                                             │
│  │ 麦克风设备  │──┐                                          │
│  └────────────┘  │     ┌──────────────────────────────┐    │
│  ┌────────────┐  │     │        效果器链处理           │    │
│  │ 线路输入   │──┼────>│  ┌─────┐ ┌─────┐ ┌─────┐   │    │
│  └────────────┘  │     │  │混响│→│合唱│→│ EQ │   │    │
│  ┌────────────┐  │     │  └─────┘ └─────┘ └─────┘   │    │
│  │ 虚拟设备   │──┘     └──────────────┬───────────────┘    │
│  └────────────┘                      │                     │
│                                      v                     │
│                              ┌──────────────┐              │
│                              │   输出路由    │              │
│                              └──────┬───────┘              │
│                    ┌───────────────┼───────────────┐       │
│                    v               v               v       │
│              [监听耳机]      [虚拟音频]      [录音输出]    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、开发阶段规划

| 阶段 | 内容 | 预计工时 |
|------|------|----------|
| **Phase 1** | 项目搭建、数据库、媒体库基础功能 | 3-4天 |
| **Phase 2** | 播放引擎、原唱/伴唱切换、歌词显示 | 4-5天 |
| **Phase 3** | 过场音乐、VAD Ducking | 2-3天 |
| **Phase 4** | 气氛组、MIDI集成 | 2-3天 |
| **Phase 5** | 音频路由、多设备输出 | 2-3天 |
| **Phase 6** | UI完善、测试、优化 | 3-4天 |
